# 윈도우 무설치 주식 모니터 (Stock Monitor) 구현 계획서 [최종 확정안]

모든 인터뷰 내용을 바탕으로 확정된 개발 계획입니다.

## 🎯 목표 & MVP (최소 기능 제품)
- **형태**: **PWA (프로그레시브 웹 앱)** 
  - 사내망 안티바이러스 우회를 위해 `.exe` 대신 웹 기반 설치형 앱을 사용합니다.
  - 작업표시줄에 고정되며, 윈도우 일반 앱처럼 단독 창으로 실행됩니다.
  - 창을 '최소화' 해두면 백그라운드에서 정상 동작합니다. (창을 완전히 닫으면 동작 중지)
- **개발 스택**: Next.js (Vercel 무료 호스팅 배포)
- **데이터 출처**: 네이버 증권 데이터 (서버 API 라우트 활용하여 CORS 우회)
- **주요 기능**:
  1. 심플한 위젯 형태의 화면 (종목 검색, 목표가 등록)
  2. 10초마다 가격 데이터 백그라운드 조회
  3. 목표가 도달 시 **윈도우 기본 토스트 알림** 및 소리 출력 후 해당 조건 자동 삭제

## 🧠 세부 로직 (의사결정 완료 사항)
- **다중 알림 등록**: **한 종목에 대해서도 여러 개의 목표가(알림)를 동시에 등록할 수 있습니다.** (예: 삼성전자 80,000원에 하나, 85,000원에 하나 각각 등록 가능)
- **목표가 도달 조건 (자동 판단)**: 사용자가 목표가를 입력하는 순간의 '현재가'를 기준으로 프로그램이 알아서 판단합니다.
  - 입력가 > 현재가: 상승 돌파 시 알림
  - 입력가 < 현재가: 하락 돌파 시 알림
- **알림 후 처리**: 알림이 울린 목표가 설정만 부분적으로 자동 삭제되어 중복 알림을 방지합니다.

## Proposed Changes

### 1. 프로젝트 초기화 (Setup)
- `npx create-next-app`로 프로젝트 생성 (Tailwind CSS 포함)
- PWA 적용을 위해 `next-pwa` 설정 추가 (Manifest 및 Service Worker 구성)

### 2. 백엔드 역할 (Next.js API Routes)
- `/api/search`: 네이버 증권 자동완성 API를 호출하여 종목명/코드 검색 결과 반환
- `/api/price`: 특정 종목 코드의 현재 가격을 스크래핑/API로 가져와 반환

### 3. 프론트엔드 (UI & Client Logic)
- **모바일/위젯 비율의 심플한 UI 디자인**
- 사용자가 종목과 여러 개의 목표가를 리스트 형태로 등록하면 로컬스토리지(Local Storage)에 저장
- `setInterval`을 이용해 10초마다 등록된 종목들의 `/api/price` 일괄 호출
- 가격 비교 후 조건 충족 시 브라우저 `Notification` API를 통한 윈도우 네이티브 알림 발생
